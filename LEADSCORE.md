# Aqua Lead Score Playbook

## 목적

- 영업사원이 리드를 받자마자 `누가`, `언제`, `무슨 액션`을 해야 하는지 바로 보이게 합니다.
- 단순 점수만 남기지 않고 `상담 유형`, `라우팅 그룹`, `액션 요약`, `Task SLA`까지 연결합니다.

## 점수 기준

### 1. 문의 목적

- `견적 요청`: 35점
- `데모 요청`: 25점
- `제품 문의`: 18점
- `기술 문의`, `파트너 문의`: 12점

### 2. 도입 시점

- `즉시`: 30점
- `1개월 이내`: 25점
- `3개월 이내`: 15점
- `6개월 이내`: 8점
- `6개월 이후`: 3점

### 3. 예산

- 고예산 구간이면 15점
- 예산 정보가 있으나 고예산까지는 아니면 8점

### 4. 고객 상황

- `경쟁사 교체`, `렌탈→구매 전환`: 15점
- `렌탈 유지 검토`, `추가 설치`: 10점
- `신규 도입`: 8점

### 5. 계약 종료 윈도우

- `종료 지남`, `30일 이내`: 20점
- `31-60일`: 12점
- `61-90일`: 6점

### 6. 규모

- 구매형/대용량/3대 이상/전환 시나리오 등 엔터프라이즈 성격이면 10점
- 2대 이상이면 5점

## 우선순위

- `Hot`: 80점 이상
- `Warm`: 55점 이상
- `Cold`: 54점 이하

## 상담 유형 자동 분류

- `파트너 문의`면 `파트너 상담`
- `렌탈→구매 전환`이면 `구매 전환 상담`
- `경쟁사 교체`면 `경쟁사 교체 상담`
- `렌탈`이고 계약 종료 윈도우가 `종료 지남`, `30일 이내`, `31-60일`, `61-90일`이면 `렌탈 갱신 상담`
- 구매형/대용량/3대 이상이면 `대형 설치 상담`
- `견적 요청`이면 `견적 상담`
- `데모 요청`이면 `데모 상담`
- `기술 문의`면 `기술 상담`
- 나머지는 `초기 상담`

## 라우팅 그룹 자동 분류

- `파트너 문의`: `파트너`
- `렌탈→구매 전환`, `경쟁사 교체`: `전략 영업`
- 구매형/대용량/3대 이상: `엔터프라이즈`
- `렌탈` + 계약 종료 윈도우 `종료 지남`, `30일 이내`, `31-60일`, `61-90일`: `리텐션`
- `견적 요청`, `데모 요청`, `Hot`: `인사이드 세일즈`
- 그 외: `인바운드`

## Task 자동화

- `Hot`: 1일 SLA Task
- `Warm`: 3일 SLA Task
- `Cold`라도 `렌탈` + 계약 종료 `31-60일`, `61-90일`이면 7일 SLA `Renewal Watch` Task

### 영업 Task 유형 표준화

- `초기 상담`: 일반 인바운드, 경쟁사 비교, 대형 설치 검토 등 기본 후속
- `견적 후속`: `견적 요청` 기반 후속
- `계약 갱신`: 렌탈 계약 종료 윈도우 기반 후속
- `구매 전환`: `렌탈→구매 전환` 시나리오
- `재공략`: `수주 실패` 이후 재오픈 가능성이 `높음` 또는 `보통`인 Opportunity 후속

### Task 설명에 자동 포함되는 정보

- 액션 유형
- 상담 유형
- 라우팅 그룹
- SLA
- 점수 / 우선순위 / 계약 종료 윈도우
- 고객 상황 / 문의 목적 / 예산 / 도입 시점 / 경쟁사 / Pain Point
- UTM / 랜딩 페이지 정보

## 현재 담당자 할당 방식

- 아직 운영용 Queue/Custom Metadata 기반 라우팅 테이블은 없습니다.
- 그래서 현재 Apex는 활성 사용자 목록을 기준으로 라우팅 그룹별 기본 분기를 적용합니다.
- `인바운드`, `리텐션`: 앞쪽 사용자
- `인사이드 세일즈`: 두 번째 우선 사용자
- `전략 영업`: 세 번째 우선 사용자
- `엔터프라이즈`, `파트너`: 마지막 사용자

운영 단계에서는 이 부분을 `Queue` 또는 `Custom Metadata`로 교체하는 것이 좋습니다.

## 영업 홈 요약 LWC

`salesRepHomeSummary`는 아래를 한 화면에서 보여줍니다.

- Hot 리드
- 지연 Task
- 갱신 관찰 리드
- 견적 후속 리드

## 다음 확장 후보

- 리드 전환 이후 Opportunity/Quote에도 동일한 `상담 유형`과 `액션 요약` 흐름 연결
- 영업 활동 기록용 커스텀 필드 또는 전용 Activity 요약 카드 추가
- Queue/팀별 실운영 라우팅 규칙을 Custom Metadata로 분리
